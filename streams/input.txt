1. What is HTML and how does it work in a webpage?
Answer: HTML (HyperText Markup Language) is the standard markup language for creating web pages. It provides the structure of web content using elements (tags) like <div>, <p>, <img>, <a>, etc. The browser parses the HTML file, builds a Document Object Model (DOM) tree, and then it can apply CSS for styling and JavaScript for behavior.
2. What are semantic HTML elements? Give examples.
Answer: Semantic HTML elements clearly describe their meaning in both human  and machine readable ways. Examples: <header>, <footer>, <nav>, <section>, <article>, <aside>. Using them helps with accessibility, SEO and maintainability.
3. What’s the difference between <div> and <span>?
Answer: <div> is a block level element—it starts on a new line and takes the full width available by default. <span> is an inline element—it does not start a new line and only takes up as much width as its content. Use <div> for large structural parts; <span> for inline pieces within text.
4. What is the difference between id and class in HTML?
Answer:
•	id is unique per page—each value should appear only once. It’s used for identifying a single element (e.g., id="main-header").
•	class can be used on multiple elements, to group them for styling or behavior (e.g., class="button primary").
IDs have higher specificity in CSS (see later).
5. How do you create a form in HTML? What are common form elements?
Answer:
<form action="/submit" method="post">
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" required>
  <input type="email" id="email" name="email">
  <button type="submit">Submit</button>
</form>
Common elements: <input> (text, password, email, checkbox, radio), <textarea>, <select> with <option>, <button>, <label>. The for attribute on <label> links to an input’s id.
6. What is the difference between <script>, <link>, and <style>?
Answer:
•	<script> is used to embed or reference JavaScript code in an HTML page.
•	<link> is used to link external resources, especially stylesheets (rel="stylesheet").
•	<style> is used to embed CSS directly in the HTML document.
Example: <link rel="stylesheet" href="styles.css">.
7. What are meta tags and why are they important?
Answer: Meta tags provide metadata about the HTML document—information not displayed directly to the user. They reside in the <head> section. Examples:
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<meta name="description" content="A description of the page">
They are important for SEO, mobile responsiveness, browser rendering, character encoding, etc.
8. What’s the difference between <section>, <article>, <aside>, and <nav>?
Answer:
•	<section> is a generic section of a document, typically with a heading.
•	<article> represents an independent piece of content (e.g., blog post, news article).
•	<aside> is for content related to the main content (sidebar, pull quote).
•	<nav> is for a section of navigation links.
Using these semantic elements improves accessibility and structure.
9. How does the browser render HTML?
Answer: The browser performs roughly these steps:
1.	Downloads HTML, builds the DOM tree.
2.	Downloads CSS and builds the CSSOM (CSS Object Model) tree.
3.	Combines DOM + CSSOM into the Render Tree.
4.	Layout: computes sizes/positions of each element.
5.	Paint: draws pixels to screen.
6.	JavaScript can modify the DOM, causing re layout and repaint.
10. What is the purpose of the alt attribute in <img>?
Answer: The alt attribute provides alternative text if the image cannot be displayed and is read by screen readers for accessibility. It helps visually impaired users and is important for SEO.

🟣 CSS Questions & Answers
1. What is the difference between class and id selectors in CSS?
Answer:
•	#myId { ... } selects the element with id="myId"; it has high specificity and usually only one element matches.
•	.myClass { ... } selects elements with class="myClass"; many elements can match.
IDs carry higher weight in the specificity algorithm than classes.
2. What are the different types of CSS? (Inline, Internal, External)
Answer:
•	Inline: <div style="color: red;"> … </div>. Applied via style attribute.
•	Internal (Embedded):
•	<style>
•	  .myClass { color: blue; }
•	</style>
In the <head>.
•	External: linked via <link rel="stylesheet" href="styles.css">. Best practice—keeping CSS separate.
3. What is the box model in CSS?
Answer: Every element is a rectangular box. It consists of:
•	Content area
•	Padding around content
•	Border around padding
•	Margin outside the border
When you set width/height, by default it applies to the content box. The actual rendered size adds padding + border (if box-sizing: content-box). Using box-sizing: border-box changes behavior to include padding and border in the width/height. W3Schools+1
4. What are pseudo‐classes and pseudo‐elements?
Answer:
•	Pseudo classes target states of elements: e.g., a:hover, .btn:active, input:focus.
•	Pseudo elements target parts of elements: e.g., p::before, p::after, span::first-letter.
They enhance styling beyond standard element selectors.
5. How does CSS specificity work?
Answer: Specificity is a calculation browsers use to decide which CSS rule applies when multiple rules could apply. It considers four parts (inline styles highest), then ID selectors, then class/attribute/pseudo‐class selectors, then element/pseudo‐element selectors. MDN Web Docs+1 For example: #nav .item p has higher specificity than .header p.
6. What is the difference between relative, absolute, fixed, and sticky positioning?
Answer:
•	position: relative; — element is positioned relative to its normal position (can be offset).
•	position: absolute; — element is removed from normal flow and placed relative to the nearest positioned ancestor (position not static).
•	position: fixed; — element is placed relative to the viewport and stays in place on scroll.
•	position: sticky; — behaves like relative until a scroll threshold is crossed, then behaves like fixed.
7. What are media queries? How do you make a page responsive?
Answer: Media queries allow CSS to apply styles conditionally based on viewport size or device characteristics. Example:
@media (max-width: 768px) {
  .sidebar { display: none; }
}
Responsive pages use flexible layouts (flex, grid), relative units (%, em, rem), and breakpoints to adjust for mobile/desktop.
8. What’s the difference between em, rem, %, and px?
Answer:
•	px: absolute pixel size.
•	%: relative to parent element’s size (for width/height) or font size for text context.
•	em: relative to the font size of the element itself.
•	rem: relative to the font size of the root element (usually <html>).
Relative units scale better across devices.
9. How do Flexbox and Grid differ? When would you use each?
Answer:
•	Flexbox is one dimensional (row or column) layout—good for aligning items along one axis.
•	Grid is two dimensional (rows and columns)—good for complex layouts where alignment across both axes matters.
Use Flexbox for things like a nav bar or card row; Grid for full page layout or irregular grid needs.
10. What is the z index and how does stacking context work?
Answer: z index controls stacking order (which elements appear on top) when elements overlap and have a position value other than static. Stacking context is created by position, opacity, transform, and other properties—only within the same context z index compares. Understanding this helps avoid unexpected layering issues.

🟡 JavaScript Questions & Answers
1. What is JavaScript and how is it different from Java?
Answer: JavaScript is a scripting language that runs in the browser (and now on servers with Node.js). It’s dynamically typed and prototype based. Java is a statically typed, object oriented language compiled to bytecode running on a JVM. Despite the similar names, they differ fundamentally.
2. What are var, let, and const?
Answer:
•	var declares a function scoped or globally scoped variable. It is hoisted (initialization to undefined).
•	let declares a block scoped variable (within {}) and is not hoisted in the same way.
•	const declares a block scoped constant (cannot be reassigned). Its contents (for objects/arrays) may be modified, but the binding cannot be reassigned.
3. What is hoisting in JavaScript?
Answer: Hoisting is JavaScript’s behavior where declarations (var, function) are moved to the top of their containing scope during compile phase. For example:
console.log(x); // undefined
var x = 5;
Because var x is hoisted, but assignment stays where it is. let and const are hoisted differently (in a temporal dead zone) so accessing them before their declaration throws a ReferenceError.
4. What are closures in JavaScript?
Answer: A closure is when an inner function retains access to its outer function’s scope even after the outer function has returned. Example:
function outer() {
  let count = 0;
  return function inner() {
    count++;
    console.log(count);
  }
}
const fn = outer();
fn(); // 1
fn(); // 2
Here inner() closes over count variable—this is a closure.
5. What is the difference between == and ===?
Answer:
•	== is equality with type conversion (coercion). Example: 5 == '5' is true.
•	=== is strict equality—both value and type must match. Example: 5 === '5' is false.
It’s best to use === to avoid unexpected coercions.
6. What is the DOM and how do you manipulate it with JavaScript?
Answer: The DOM (Document Object Model) is the tree like representation of the HTML document elements and their properties. With JavaScript you can manipulate it:
const btn = document.getElementById("myBtn");
btn.addEventListener("click", () => {
  document.getElementById("text").textContent = "Clicked!";
});
You can select elements (getElementById, querySelector), change attributes, add/remove elements, listen to events, etc.
7. Explain event bubbling and event delegation.
Answer:
•	Event bubbling: When an event occurs on an element, it first runs its handlers, then the event moves up (bubbles) to ancestor elements, triggering their handlers.
•	Event delegation: Instead of attaching handler to many child elements, you attach a handler to a parent and use the event target to determine which child triggered the event. This improves performance and code brevity.
8. What is the difference between synchronous and asynchronous code?
Answer:
•	Synchronous code runs line by line, blocking further execution until current line finishes.
•	Asynchronous code allows other work to happen while waiting (for I/O, network). In JS you use callbacks, promises, async/await to handle asynchronous behavior.
9. What are Promises? How is async/await used?
Answer:
A Promise is an object that represents eventual completion or failure of asynchronous operation. Example:
const promise = new Promise((resolve, reject) => {
  setTimeout(() => resolve("done"), 1000);
});
promise.then(result => console.log(result));
async/await provides syntactic sugar:
async function getData() {
  try {
    const result = await fetch("/api");
    const data = await result.json();
    console.log(data);
  } catch (err) {
    console.error(err);
  }
}
It makes asynchronous code look synchronous.
10. What are arrow functions? How are they different from regular functions?
Answer:
Arrow functions (()=>{}) are shorthand syntax introduced in ES6. Differences:
•	They don’t have their own this, arguments, super, or new.target context—they use the surrounding lexical this.
•	They cannot be used as constructors (you can’t use new ArrowFunc()).
Example:
const add = (a,b) => a + b;

-Answer: It is a distributed version control system (VCS) designed to track changes in files, especially source code, and facilitate collaboration among developers.
-GitHub is a web-based platform that hosts Git repositories, providing developers with tools for version control and collaboration.  GitHub uses Git, a distributed version control system that tracks changes to your code. Git allows multiple developers to work on a project simultaneously without overwriting each other's changes. It keeps a history of all changes, making it easy to revert to previous versions if needed.

3. What is hoisting in JavaScript?
Answer: Hoisting is JavaScript’s behavior where declarations (var, function) are moved to the top of their containing scope during compile phase. For example:
console.log(x); // undefined
var x = 5;
Because var x is hoisted, but assignment stays where it is. let and const are hoisted differently (in a temporal dead zone) so accessing them before their declaration throws a ReferenceError.
4. What are closures in JavaScript?
Answer: A closure is when an inner function retains access to its outer function’s scope even after the outer function has returned. Example:
function outer() {
  let count = 0;
  return function inner() {
    count++;
    console.log(count);
  }
}
const fn = outer();
fn(); // 1
fn(); // 2
Here inner() closes over count variable—this is a closure.
5. What is the difference between == and ===?
Answer:
•	== is equality with type conversion (coercion). Example: 5 == '5' is true.
•	=== is strict equality—both value and type must match. Example: 5 === '5' is false.
It’s best to use === to avoid unexpected coercions.
6. What is the DOM and how do you manipulate it with JavaScript?
Answer: The DOM (Document Object Model) is the tree like representation of the HTML document elements and their properties. With JavaScript you can manipulate it:
const btn = document.getElementById("myBtn");

  document.getElementById("text").textContent = "Clicked!";

You can select elements (getElementById, querySelector), change attributes, add/remove elements, listen to events, etc.
7. Explain event bubbling and event delegation.
Answer:
•	Event bubbling: When an event occurs on an element, it first runs its handlers, then the event moves up (bubbles) to ancestor elements, triggering their handlers.
•	Event delegation: Instead of attaching handler to many child elements, you attach a handler to a parent and use the event target to determine which child triggered the event. This improves performance and code brevity.
8. What is the difference between synchronous and asynchronous code?
Answer:
•	Synchronous code runs line by line, blocking further execution until current line finishes.
•	Asynchronous code allows other work to happen while waiting (for I/O, network). In JS you use callbacks, promises, async/await to handle asynchronous behavior.
9. What are Promises? How is async/await used?
Answer:

const promise = new Promise((resolve, reject) => {
  setTimeout(() => resolve("done"), 1000);
});
promise.then(result => console.log(result));
async/await provides syntactic sugar:
async function getData() {
  try {

  } catch (err) {
    console.error(err);
  }
}
It makes asynchronous code look synchronous.
10. What are arrow functions? How are they different from regular functions?
Answer:

•	They don’t have their own this, arguments, super, or new.target context—they use the surrounding lexical this.
•	They cannot be used as constructors (you can’t use new ArrowFunc()).
Example:
const add = (a,b) => a + b;

It provides the structure of web content using elements (tags) like <div>, <p>, <img>, <a>, etc. The browser parses the HTML file, builds a Document Object Model (DOM) tree, and then it can apply CSS for styling and JavaScript for behavior.
2. What are semantic HTML elements? Give examples.
Answer: Semantic HTML elements clearly describe their meaning in both human  and machine readable ways. Examples: <header>, <footer>, <nav>, <section>, <article>, <aside>. Using them helps with accessibility, SEO and maintainability.
3. What’s the difference between <div> and <span>?
Answer: <div> is a block level element—it starts on a new line and takes the full width available by default. <span> is an inline element—it does not start a new line and only takes up as much width as its content. Use <div> for large structural parts; <span> for inline pieces within text.
4. What is the difference between id and class in HTML?
Answer:
•	id is unique per page—each value should appear only once. It’s used for identifying a single element (e.g., id="main-header").
•	class can be used on multiple elements, to group them for styling or behavior (e.g., class="button primary").
IDs have higher specificity in CSS (see later).
5. How do you create a form in HTML? What are common form elements?
Answer:
<form action="/submit" method="post">
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" required>
  <input type="email" id="email" name="email">
  <button type="submit">Submit</button>
</form>
Common elements: <input> (text, password, email, checkbox, radio), <textarea>, <select> with <option>, <button>, <label>. The for attribute on <label> links to an input’s id.
6. What is the difference between <script>, <link>, and <style>?
Answer:
•	<script> is used to embed or reference JavaScript code in an HTML page.
•	<link> is used to link external resources, especially stylesheets (rel="stylesheet").
•	<style> is used to embed CSS directly in the HTML document.
Example: <link rel="stylesheet" href="styles.css">.
7. What are meta tags and why are they important?
Answer: Meta tags provide metadata about the HTML document—information not displayed directly to the user. They reside in the <head> section. Examples:
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<meta name="description" content="A description of the page">
They are important for SEO, mobile responsiveness, browser rendering, character encoding, etc.
8. What’s the difference between <section>, <article>, <aside>, and <nav>?
Answer:
•	<section> is a generic section of a document, typically with a heading.
•	<article> represents an independent piece of content (e.g., blog post, news article).
•	<aside> is for content related to the main content (sidebar, pull quote).
•	<nav> is for a section of navigation links.
Using these semantic elements improves accessibility and structure.
9. How does the browser render HTML?
Answer: The browser performs roughly these steps:
1.	Downloads HTML, builds the DOM tree.
2.	Downloads CSS and builds the CSSOM (CSS Object Model) tree.
3.	Combines DOM + CSSOM into the Render Tree.
4.	Layout: computes sizes/positions of each element.
5.	Paint: draws pixels to screen.
6.	JavaScript can modify the DOM, causing re layout and repaint.
10. What is the purpose of the alt attribute in <img>?
Answer: The alt attribute provides alternative text if the image cannot be displayed and is read by screen readers for accessibility. It helps visually impaired users and is important for SEO.

🟣 CSS Questions & Answers
1. What is the difference between class and id selectors in CSS?
Answer:
•	#myId { ... } selects the element with id="myId"; it has high specificity and usually only one element matches.
•	.myClass { ... } selects elements with class="myClass"; many elements can match.
IDs carry higher weight in the specificity algorithm than classes.
2. What are the different types of CSS? (Inline, Internal, External)
Answer:
•	Inline: <div style="color: red;"> … </div>. Applied via style attribute.
•	Internal (Embedded):
•	<style>
•	  .myClass { color: blue; }
•	</style>
In the <head>.
•	External: linked via <link rel="stylesheet" href="styles.css">. Best practice—keeping CSS separate.
3. What is the box model in CSS?
Answer: Every element is a rectangular box. It consists of:
•	Content area
•	Padding around content
•	Border around padding
•	Margin outside the border
When you set width/height, by default it applies to the content box. The actual rendered size adds padding + border (if box-sizing: content-box). Using box-sizing: border-box changes behavior to include padding and border in the width/height. W3Schools+1
4. What are pseudo‐classes and pseudo‐elements?
Answer:
•	Pseudo classes target states of elements: e.g., a:hover, .btn:active, input:focus.
•	Pseudo elements target parts of elements: e.g., p::before, p::after, span::first-letter.
They enhance styling beyond standard element selectors.
5. How does CSS specificity work?
Answer: Specificity is a calculation browsers use to decide which CSS rule applies when multiple rules could apply. It considers four parts (inline styles highest), then ID selectors, then class/attribute/pseudo‐class selectors, then element/pseudo‐element selectors. MDN Web Docs+1 For example: #nav .item p has higher specificity than .header p.
6. What is the difference between relative, absolute, fixed, and sticky positioning?
Answer:
•	position: relative; — element is positioned relative to its normal position (can be offset).
•	position: absolute; — element is removed from normal flow and placed relative to the nearest positioned ancestor (position not static).
•	position: fixed; — element is placed relative to the viewport and stays in place on scroll.
•	position: sticky; — behaves like relative until a scroll threshold is crossed, then behaves like fixed.
7. What are media queries? How do you make a page responsive?
Answer: Media queries allow CSS to apply styles conditionally based on viewport size or device characteristics. Example:
@media (max-width: 768px) {
  .sidebar { display: none; }
}
Responsive pages use flexible layouts (flex, grid), relative units (%, em, rem), and breakpoints to adjust for mobile/desktop.
8. What’s the difference between em, rem, %, and px?
Answer:
•	px: absolute pixel size.
•	%: relative to parent element’s size (for width/height) or font size for text context.
•	em: relative to the font size of the element itself.
•	rem: relative to the font size of the root element (usually <html>).
Relative units scale better across devices.
9. How do Flexbox and Grid differ? When would you use each?
Answer:
•	Flexbox is one dimensional (row or column) layout—good for aligning items along one axis.
•	Grid is two dimensional (rows and columns)—good for complex layouts where alignment across both axes matters.
Use Flexbox for things like a nav bar or card row; Grid for full page layout or irregular grid needs.
10. What is the z index and how does stacking context work?
Answer: z index controls stacking order (which elements appear on top) when elements overlap and have a position value other than static. Stacking context is created by position, opacity, transform, and other properties—only within the same context z index compares. Understanding this helps avoid unexpected layering issues.

🟡 JavaScript Questions & Answers
1. What is JavaScript and how is it different from Java?
Answer: JavaScript is a scripting language that runs in the browser (and now on servers with Node.js). It’s dynamically typed and prototype based. Java is a statically typed, object oriented language compiled to bytecode running on a JVM. Despite the similar names, they differ fundamentally.
2. What are var, let, and const?
Answer:
•	var declares a function scoped or globally scoped variable. It is hoisted (initialization to undefined).
•	let declares a block scoped variable (within {}) and is not hoisted in the same way.
•	const declares a block scoped constant (cannot be reassigned). Its contents (for objects/arrays) may be modified, but the binding cannot be reassigned.
3. What is hoisting in JavaScript?
Answer: Hoisting is JavaScript’s behavior where declarations (var, function) are moved to the top of their containing scope during compile phase. For example:
console.log(x); // undefined
var x = 5;
Because var x is hoisted, but assignment stays where it is. let and const are hoisted differently (in a temporal dead zone) so accessing them before their declaration throws a ReferenceError.
4. What are closures in JavaScript?
Answer: A closure is when an inner function retains access to its outer function’s scope even after the outer function has returned. Example:
function outer() {
  let count = 0;
  return function inner() {
    count++;
    console.log(count);
  }
}
const fn = outer();
fn(); // 1
fn(); // 2
Here inner() closes over count variable—this is a closure.
5. What is the difference between == and ===?
Answer:
•	== is equality with type conversion (coercion). Example: 5 == '5' is true.
•	=== is strict equality—both value and type must match. Example: 5 === '5' is false.
It’s best to use === to avoid unexpected coercions.
6. What is the DOM and how do you manipulate it with JavaScript?
Answer: The DOM (Document Object Model) is the tree like representation of the HTML document elements and their properties. With JavaScript you can manipulate it:
const btn = document.getElementById("myBtn");
btn.addEventListener("click", () => {
  document.getElementById("text").textContent = "Clicked!";
});
You can select elements (getElementById, querySelector), change attributes, add/remove elements, listen to events, etc.
7. Explain event bubbling and event delegation.
Answer:
•	Event bubbling: When an event occurs on an element, it first runs its handlers, then the event moves up (bubbles) to ancestor elements, triggering their handlers.
•	Event delegation: Instead of attaching handler to many child elements, you attach a handler to a parent and use the event target to determine which child triggered the event. This improves performance and code brevity.
8. What is the difference between synchronous and asynchronous code?
Answer:
•	Synchronous code runs line by line, blocking further execution until current line finishes.
•	Asynchronous code allows other work to happen while waiting (for I/O, network). In JS you use callbacks, promises, async/await to handle asynchronous behavior.
9. What are Promises? How is async/await used?
Answer:
A Promise is an object that represents eventual completion or failure of asynchronous operation. Example:
const promise = new Promise((resolve, reject) => {
  setTimeout(() => resolve("done"), 1000);
});
promise.then(result => console.log(result));
async/await provides syntactic sugar:
async function getData() {
  try {
    const result = await fetch("/api");
    const data = await result.json();
    console.log(data);
  } catch (err) {
    console.error(err);
  }
}
It makes asynchronous code look synchronous.
10. What are arrow functions? How are they different from regular functions?
Answer:
Arrow functions (()=>{}) are shorthand syntax introduced in ES6. Differences:
•	They don’t have their own this, arguments, super, or new.target context—they use the surrounding lexical this.
•	They cannot be used as constructors (you can’t use new ArrowFunc()).
Example:
const add = (a,b) => a + b;

-Answer: It is a distributed version control system (VCS) designed to track changes in files, especially source code, and facilitate collaboration among developers.
-GitHub is a web-based platform that hosts Git repositories, providing developers with tools for version control and collaboration.  GitHub uses Git, a distributed version control system that tracks changes to your code. Git allows multiple developers to work on a project simultaneously without overwriting each other's changes. It keeps a history of all changes, making it easy to revert to previous versions if needed.

3. What is hoisting in JavaScript?
Answer: Hoisting is JavaScript’s behavior where declarations (var, function) are moved to the top of their containing scope during compile phase. For example:
console.log(x); // undefined
var x = 5;
Because var x is hoisted, but assignment stays where it is. let and const are hoisted differently (in a temporal dead zone) so accessing them before their declaration throws a ReferenceError.
4. What are closures in JavaScript?
Answer: A closure is when an inner function retains access to its outer function’s scope even after the outer function has returned. Example:
function outer() {
  let count = 0;
  return function inner() {
    count++;
    console.log(count);
  }
}
const fn = outer();
fn(); // 1
fn(); // 2
Here inner() closes over count variable—this is a closure.
5. What is the difference between == and ===?
Answer:
•	== is equality with type conversion (coercion). Example: 5 == '5' is true.
•	=== is strict equality—both value and type must match. Example: 5 === '5' is false.
It’s best to use === to avoid unexpected coercions.
6. What is the DOM and how do you manipulate it with JavaScript?
Answer: The DOM (Document Object Model) is the tree like representation of the HTML document elements and their properties. With JavaScript you can manipulate it:
const btn = document.getElementById("myBtn");

  document.getElementById("text").textContent = "Clicked!";

You can select elements (getElementById, querySelector), change attributes, add/remove elements, listen to events, etc.
7. Explain event bubbling and event delegation.
Answer:
•	Event bubbling: When an event occurs on an element, it first runs its handlers, then the event moves up (bubbles) to ancestor elements, triggering their handlers.
•	Event delegation: Instead of attaching handler to many child elements, you attach a handler to a parent and use the event target to determine which child triggered the event. This improves performance and code brevity.
8. What is the difference between synchronous and asynchronous code?
Answer:
•	Synchronous code runs line by line, blocking further execution until current line finishes.
•	Asynchronous code allows other work to happen while waiting (for I/O, network). In JS you use callbacks, promises, async/await to handle asynchronous behavior.
9. What are Promises? How is async/await used?
Answer:

const promise = new Promise((resolve, reject) => {
  setTimeout(() => resolve("done"), 1000);
});
promise.then(result => console.log(result));
async/await provides syntactic sugar:
async function getData() {
  try {

    const data = await result.json();
    console.log(data);
  } catch (err) {
    console.error(err);
  }
}
It makes asynchronous code look synchronous.
10. What are arrow functions? How are they different from regular functions?
Answer:

•	They don’t have their own this, arguments, super, or new.target context—they use the surrounding lexical this.
•	They cannot be used as constructors (you can’t use new ArrowFunc()).
Example:
const add = (a,b) => a + b;

3. What’s the difference between <div> and <span>?
Answer: <div> is a block level element—it starts on a new line and takes the full width available by default. <span> is an inline element—it does not start a new line and only takes up as much width as its content. Use <div> for large structural parts; <span> for inline pieces within text.

Answer:
•	id is unique per page—each value should appear only once. It’s used for identifying a single element (e.g., id="main-header").

•	<article> represents an independent piece of content (e.g., blog post, news article).

It provides the structure of web content using elements (tags) like <div>, <p>, <img>, <a>, etc. The browser parses the HTML file, builds a Document Object Model (DOM) tree, and then it can apply CSS for styling and JavaScript for behavior.
2. What are semantic HTML elements? Give examples.
Answer: Semantic HTML elements clearly describe their meaning in both human  and machine readable ways. Examples: <header>, <footer>, <nav>, <section>, <article>, <aside>. Using them helps with accessibility, SEO and maintainability.
3. What’s the difference between <div> and <span>?
Answer: <div> is a block level element—it starts on a new line and takes the full width available by default. <span> is an inline element—it does not start a new line and only takes up as much width as its content. Use <div> for large structural parts; <span> for inline pieces within text.
4. What is the difference between id and class in HTML?
Answer:
•	id is unique per page—each value should appear only once. It’s used for identifying a single element (e.g., id="main-header").
•	class can be used on multiple elements, to group them for styling or behavior (e.g., class="button primary").
IDs have higher specificity in CSS (see later).
5. How do you create a form in HTML? What are common form elements?
Answer:
<form action="/submit" method="post">
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" required>
  <input type="email" id="email" name="email">
  <button type="submit">Submit</button>
</form>
Common elements: <input> (text, password, email, checkbox, radio), <textarea>, <select> with <option>, <button>, <label>. The for attribute on <label> links to an input’s id.
6. What is the difference between <script>, <link>, and <style>?
Answer:
•	<script> is used to embed or reference JavaScript code in an HTML page.
•	<link> is used to link external resources, especially stylesheets (rel="stylesheet").
•	<style> is used to embed CSS directly in the HTML document.
Example: <link rel="stylesheet" href="styles.css">.
7. What are meta tags and why are they important?
Answer: Meta tags provide metadata about the HTML document—information not displayed directly to the user. They reside in the <head> section. Examples:
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<meta name="description" content="A description of the page">
They are important for SEO, mobile responsiveness, browser rendering, character encoding, etc.
8. What’s the difference between <section>, <article>, <aside>, and <nav>?
Answer:
•	<section> is a generic section of a document, typically with a heading.
•	<article> represents an independent piece of content (e.g., blog post, news article).
•	<aside> is for content related to the main content (sidebar, pull quote).
•	<nav> is for a section of navigation links.
Using these semantic elements improves accessibility and structure.
9. How does the browser render HTML?
Answer: The browser performs roughly these steps:
1.	Downloads HTML, builds the DOM tree.
2.	Downloads CSS and builds the CSSOM (CSS Object Model) tree.
3.	Combines DOM + CSSOM into the Render Tree.
4.	Layout: computes sizes/positions of each element.
5.	Paint: draws pixels to screen.
6.	JavaScript can modify the DOM, causing re layout and repaint.
10. What is the purpose of the alt attribute in <img>?
Answer: The alt attribute provides alternative text if the image cannot be displayed and is read by screen readers for accessibility. It helps visually impaired users and is important for SEO.

🟣 CSS Questions & Answers
1. What is the difference between class and id selectors in CSS?
Answer:
•	#myId { ... } selects the element with id="myId"; it has high specificity and usually only one element matches.
•	.myClass { ... } selects elements with class="myClass"; many elements can match.
IDs carry higher weight in the specificity algorithm than classes.
2. What are the different types of CSS? (Inline, Internal, External)
Answer:
•	Inline: <div style="color: red;"> … </div>. Applied via style attribute.
•	Internal (Embedded):
•	<style>
•	  .myClass { color: blue; }
•	</style>
In the <head>.
•	External: linked via <link rel="stylesheet" href="styles.css">. Best practice—keeping CSS separate.
3. What is the box model in CSS?
Answer: Every element is a rectangular box. It consists of:
•	Content area
•	Padding around content
•	Border around padding
•	Margin outside the border
When you set width/height, by default it applies to the content box. The actual rendered size adds padding + border (if box-sizing: content-box). Using box-sizing: border-box changes behavior to include padding and border in the width/height. W3Schools+1
4. What are pseudo‐classes and pseudo‐elements?
Answer:
•	Pseudo classes target states of elements: e.g., a:hover, .btn:active, input:focus.
•	Pseudo elements target parts of elements: e.g., p::before, p::after, span::first-letter.
They enhance styling beyond standard element selectors.
5. How does CSS specificity work?
Answer: Specificity is a calculation browsers use to decide which CSS rule applies when multiple rules could apply. It considers four parts (inline styles highest), then ID selectors, then class/attribute/pseudo‐class selectors, then element/pseudo‐element selectors. MDN Web Docs+1 For example: #nav .item p has higher specificity than .header p.
6. What is the difference between relative, absolute, fixed, and sticky positioning?
Answer:
•	position: relative; — element is positioned relative to its normal position (can be offset).
•	position: absolute; — element is removed from normal flow and placed relative to the nearest positioned ancestor (position not static).
•	position: fixed; — element is placed relative to the viewport and stays in place on scroll.
•	position: sticky; — behaves like relative until a scroll threshold is crossed, then behaves like fixed.
7. What are media queries? How do you make a page responsive?
Answer: Media queries allow CSS to apply styles conditionally based on viewport size or device characteristics. Example:
@media (max-width: 768px) {
  .sidebar { display: none; }
}
Responsive pages use flexible layouts (flex, grid), relative units (%, em, rem), and breakpoints to adjust for mobile/desktop.
8. What’s the difference between em, rem, %, and px?
Answer:
•	px: absolute pixel size.
•	%: relative to parent element’s size (for width/height) or font size for text context.
•	em: relative to the font size of the element itself.
•	rem: relative to the font size of the root element (usually <html>).
Relative units scale better across devices.
9. How do Flexbox and Grid differ? When would you use each?
Answer:
•	Flexbox is one dimensional (row or column) layout—good for aligning items along one axis.
•	Grid is two dimensional (rows and columns)—good for complex layouts where alignment across both axes matters.
Use Flexbox for things like a nav bar or card row; Grid for full page layout or irregular grid needs.
10. What is the z index and how does stacking context work?
Answer: z index controls stacking order (which elements appear on top) when elements overlap and have a position value other than static. Stacking context is created by position, opacity, transform, and other properties—only within the same context z index compares. Understanding this helps avoid unexpected layering issues.

🟡 JavaScript Questions & Answers
1. What is JavaScript and how is it different from Java?
Answer: JavaScript is a scripting language that runs in the browser (and now on servers with Node.js). It’s dynamically typed and prototype based. Java is a statically typed, object oriented language compiled to bytecode running on a JVM. Despite the similar names, they differ fundamentally.
2. What are var, let, and const?
Answer:
•	var declares a function scoped or globally scoped variable. It is hoisted (initialization to undefined).
•	let declares a block scoped variable (within {}) and is not hoisted in the same way.
•	const declares a block scoped constant (cannot be reassigned). Its contents (for objects/arrays) may be modified, but the binding cannot be reassigned.
3. What is hoisting in JavaScript?
Answer: Hoisting is JavaScript’s behavior where declarations (var, function) are moved to the top of their containing scope during compile phase. For example:
console.log(x); // undefined
var x = 5;
Because var x is hoisted, but assignment stays where it is. let and const are hoisted differently (in a temporal dead zone) so accessing them before their declaration throws a ReferenceError.
4. What are closures in JavaScript?
Answer: A closure is when an inner function retains access to its outer function’s scope even after the outer function has returned. Example:
function outer() {
  let count = 0;
  return function inner() {
    count++;
    console.log(count);
  }
}
const fn = outer();
fn(); // 1
fn(); // 2
Here inner() closes over count variable—this is a closure.
5. What is the difference between == and ===?
Answer:
•	== is equality with type conversion (coercion). Example: 5 == '5' is true.
•	=== is strict equality—both value and type must match. Example: 5 === '5' is false.
It’s best to use === to avoid unexpected coercions.
6. What is the DOM and how do you manipulate it with JavaScript?
Answer: The DOM (Document Object Model) is the tree like representation of the HTML document elements and their properties. With JavaScript you can manipulate it:
const btn = document.getElementById("myBtn");
btn.addEventListener("click", () => {
  document.getElementById("text").textContent = "Clicked!";
});
You can select elements (getElementById, querySelector), change attributes, add/remove elements, listen to events, etc.
7. Explain event bubbling and event delegation.
Answer:
•	Event bubbling: When an event occurs on an element, it first runs its handlers, then the event moves up (bubbles) to ancestor elements, triggering their handlers.
•	Event delegation: Instead of attaching handler to many child elements, you attach a handler to a parent and use the event target to determine which child triggered the event. This improves performance and code brevity.
8. What is the difference between synchronous and asynchronous code?
Answer:
•	Synchronous code runs line by line, blocking further execution until current line finishes.
•	Asynchronous code allows other work to happen while waiting (for I/O, network). In JS you use callbacks, promises, async/await to handle asynchronous behavior.
9. What are Promises? How is async/await used?
Answer:
A Promise is an object that represents eventual completion or failure of asynchronous operation. Example:
const promise = new Promise((resolve, reject) => {
  setTimeout(() => resolve("done"), 1000);
});
promise.then(result => console.log(result));
async/await provides syntactic sugar:
async function getData() {
  try {
    const result = await fetch("/api");
    const data = await result.json();
    console.log(data);
  } catch (err) {
    console.error(err);
  }
}
It makes asynchronous code look synchronous.
10. What are arrow functions? How are they different from regular functions?
Answer:
Arrow functions (()=>{}) are shorthand syntax introduced in ES6. Differences:
•	They don’t have their own this, arguments, super, or new.target context—they use the surrounding lexical this.
•	They cannot be used as constructors (you can’t use new ArrowFunc()).
Example:
const add = (a,b) => a + b;

-Answer: It is a distributed version control system (VCS) designed to track changes in files, especially source code, and facilitate collaboration among developers.
-GitHub is a web-based platform that hosts Git repositories, providing developers with tools for version control and collaboration.  GitHub uses Git, a distributed version control system that tracks changes to your code. Git allows multiple developers to work on a project simultaneously without overwriting each other's changes. It keeps a history of all changes, making it easy to revert to previous versions if needed.

3. What is hoisting in JavaScript?
Answer: Hoisting is JavaScript’s behavior where declarations (var, function) are moved to the top of their containing scope during compile phase. For example:
console.log(x); // undefined
var x = 5;
Because var x is hoisted, but assignment stays where it is. let and const are hoisted differently (in a temporal dead zone) so accessing them before their declaration throws a ReferenceError.
4. What are closures in JavaScript?
Answer: A closure is when an inner function retains access to its outer function’s scope even after the outer function has returned. Example:
function outer() {
  let count = 0;
  return function inner() {
    count++;
    console.log(count);
  }
}
const fn = outer();
fn(); // 1
fn(); // 2
Here inner() closes over count variable—this is a closure.
5. What is the difference between == and ===?
Answer:
•	== is equality with type conversion (coercion). Example: 5 == '5' is true.
•	=== is strict equality—both value and type must match. Example: 5 === '5' is false.
It’s best to use === to avoid unexpected coercions.
6. What is the DOM and how do you manipulate it with JavaScript?
Answer: The DOM (Document Object Model) is the tree like representation of the HTML document elements and their properties. With JavaScript you can manipulate it:
const btn = document.getElementById("myBtn");

  document.getElementById("text").textContent = "Clicked!";

You can select elements (getElementById, querySelector), change attributes, add/remove elements, listen to events, etc.
7. Explain event bubbling and event delegation.
Answer:
•	Event bubbling: When an event occurs on an element, it first runs its handlers, then the event moves up (bubbles) to ancestor elements, triggering their handlers.
•	Event delegation: Instead of attaching handler to many child elements, you attach a handler to a parent and use the event target to determine which child triggered the event. This improves performance and code brevity.
8. What is the difference between synchronous and asynchronous code?
Answer:
•	Synchronous code runs line by line, blocking further execution until current line finishes.
•	Asynchronous code allows other work to happen while waiting (for I/O, network). In JS you use callbacks, promises, async/await to handle asynchronous behavior.
9. What are Promises? How is async/await used?
Answer:

const promise = new Promise((resolve, reject) => {
  setTimeout(() => resolve("done"), 1000);
});
promise.then(result => console.log(result));
async/await provides syntactic sugar:
async function getData() {

  } catch (err) {
    console.error(err);
  }
}
It makes asynchronous code look synchronous.
10. What are arrow functions? How are they different from regular functions?
Answer:

•	They don’t have their own this, arguments, super, or new.target context—they use the surrounding lexical this.
•	They cannot be used as constructors (you can’t use new ArrowFunc()).
Example:
const add = (a,b) => a + b;

3. What’s the difference between <div> and <span>?
Answer: <div> is a block level element—it starts on a new line and takes the full width available by default. <span> is an inline element—it does not start a new line and only takes up as much width as its content. Use <div> for large structural parts; <span> for inline pieces within text.
4. What is the difference between id and class in HTML?
Answer:
•	id is unique per page—each value should appear only once. It’s used for identifying a single element (e.g., id="main-header").

•	<article> represents an independent piece of content (e.g., blog post, news article).
•	<aside> is for content related to the main content (sidebar, pull quote).
•	<nav> is for a section of navigation links.

5.	Paint: draws pixels to screen.

It provides the structure of web content using elements (tags) like <div>, <p>, <img>, <a>, etc. The browser parses the HTML file, builds a Document Object Model (DOM) tree, and then it can apply CSS for styling and JavaScript for behavior.
2. What are semantic HTML elements? Give examples.
Answer: Semantic HTML elements clearly describe their meaning in both human  and machine readable ways. Examples: <header>, <footer>, <nav>, <section>, <article>, <aside>. Using them helps with accessibility, SEO and maintainability.
3. What’s the difference between <div> and <span>?
Answer: <div> is a block level element—it starts on a new line and takes the full width available by default. <span> is an inline element—it does not start a new line and only takes up as much width as its content. Use <div> for large structural parts; <span> for inline pieces within text.
4. What is the difference between id and class in HTML?
Answer:
•	id is unique per page—each value should appear only once. It’s used for identifying a single element (e.g., id="main-header").
•	class can be used on multiple elements, to group them for styling or behavior (e.g., class="button primary").
IDs have higher specificity in CSS (see later).
5. How do you create a form in HTML? What are common form elements?
Answer:
<form action="/submit" method="post">
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" required>
  <input type="email" id="email" name="email">
  <button type="submit">Submit</button>
</form>
Common elements: <input> (text, password, email, checkbox, radio), <textarea>, <select> with <option>, <button>, <label>. The for attribute on <label> links to an input’s id.
6. What is the difference between <script>, <link>, and <style>?
Answer:
•	<script> is used to embed or reference JavaScript code in an HTML page.
•	<link> is used to link external resources, especially stylesheets (rel="stylesheet").
•	<style> is used to embed CSS directly in the HTML document.
Example: <link rel="stylesheet" href="styles.css">.
7. What are meta tags and why are they important?
Answer: Meta tags provide metadata about the HTML document—information not displayed directly to the user. They reside in the <head> section. Examples:
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<meta name="description" content="A description of the page">
They are important for SEO, mobile responsiveness, browser rendering, character encoding, etc.
8. What’s the difference between <section>, <article>, <aside>, and <nav>?
Answer:
•	<section> is a generic section of a document, typically with a heading.
•	<article> represents an independent piece of content (e.g., blog post, news article).
•	<aside> is for content related to the main content (sidebar, pull quote).
•	<nav> is for a section of navigation links.
Using these semantic elements improves accessibility and structure.
9. How does the browser render HTML?
Answer: The browser performs roughly these steps:
1.	Downloads HTML, builds the DOM tree.
2.	Downloads CSS and builds the CSSOM (CSS Object Model) tree.
3.	Combines DOM + CSSOM into the Render Tree.
4.	Layout: computes sizes/positions of each element.
5.	Paint: draws pixels to screen.
6.	JavaScript can modify the DOM, causing re layout and repaint.
10. What is the purpose of the alt attribute in <img>?
Answer: The alt attribute provides alternative text if the image cannot be displayed and is read by screen readers for accessibility. It helps visually impaired users and is important for SEO.

🟣 CSS Questions & Answers
1. What is the difference between class and id selectors in CSS?
Answer:
•	#myId { ... } selects the element with id="myId"; it has high specificity and usually only one element matches.
•	.myClass { ... } selects elements with class="myClass"; many elements can match.
IDs carry higher weight in the specificity algorithm than classes.
2. What are the different types of CSS? (Inline, Internal, External)
Answer:
•	Inline: <div style="color: red;"> … </div>. Applied via style attribute.
•	Internal (Embedded):
•	<style>
•	  .myClass { color: blue; }
•	</style>
In the <head>.
•	External: linked via <link rel="stylesheet" href="styles.css">. Best practice—keeping CSS separate.
3. What is the box model in CSS?
Answer: Every element is a rectangular box. It consists of:
•	Content area
•	Padding around content
•	Border around padding
•	Margin outside the border
When you set width/height, by default it applies to the content box. The actual rendered size adds padding + border (if box-sizing: content-box). Using box-sizing: border-box changes behavior to include padding and border in the width/height. W3Schools+1
4. What are pseudo‐classes and pseudo‐elements?
Answer:
•	Pseudo classes target states of elements: e.g., a:hover, .btn:active, input:focus.
•	Pseudo elements target parts of elements: e.g., p::before, p::after, span::first-letter.
They enhance styling beyond standard element selectors.
5. How does CSS specificity work?
Answer: Specificity is a calculation browsers use to decide which CSS rule applies when multiple rules could apply. It considers four parts (inline styles highest), then ID selectors, then class/attribute/pseudo‐class selectors, then element/pseudo‐element selectors. MDN Web Docs+1 For example: #nav .item p has higher specificity than .header p.
6. What is the difference between relative, absolute, fixed, and sticky positioning?
Answer:
•	position: relative; — element is positioned relative to its normal position (can be offset).
•	position: absolute; — element is removed from normal flow and placed relative to the nearest positioned ancestor (position not static).
•	position: fixed; — element is placed relative to the viewport and stays in place on scroll.
•	position: sticky; — behaves like relative until a scroll threshold is crossed, then behaves like fixed.
7. What are media queries? How do you make a page responsive?
Answer: Media queries allow CSS to apply styles conditionally based on viewport size or device characteristics. Example:
@media (max-width: 768px) {
  .sidebar { display: none; }
}
Responsive pages use flexible layouts (flex, grid), relative units (%, em, rem), and breakpoints to adjust for mobile/desktop.
8. What’s the difference between em, rem, %, and px?
Answer:
•	px: absolute pixel size.
•	%: relative to parent element’s size (for width/height) or font size for text context.
•	em: relative to the font size of the element itself.
•	rem: relative to the font size of the root element (usually <html>).
Relative units scale better across devices.
9. How do Flexbox and Grid differ? When would you use each?
Answer:
•	Flexbox is one dimensional (row or column) layout—good for aligning items along one axis.
•	Grid is two dimensional (rows and columns)—good for complex layouts where alignment across both axes matters.
Use Flexbox for things like a nav bar or card row; Grid for full page layout or irregular grid needs.
10. What is the z index and how does stacking context work?
Answer: z index controls stacking order (which elements appear on top) when elements overlap and have a position value other than static. Stacking context is created by position, opacity, transform, and other properties—only within the same context z index compares. Understanding this helps avoid unexpected layering issues.

🟡 JavaScript Questions & Answers
1. What is JavaScript and how is it different from Java?
Answer: JavaScript is a scripting language that runs in the browser (and now on servers with Node.js). It’s dynamically typed and prototype based. Java is a statically typed, object oriented language compiled to bytecode running on a JVM. Despite the similar names, they differ fundamentally.
2. What are var, let, and const?
Answer:
•	var declares a function scoped or globally scoped variable. It is hoisted (initialization to undefined).
•	let declares a block scoped variable (within {}) and is not hoisted in the same way.
•	const declares a block scoped constant (cannot be reassigned). Its contents (for objects/arrays) may be modified, but the binding cannot be reassigned.
3. What is hoisting in JavaScript?
Answer: Hoisting is JavaScript’s behavior where declarations (var, function) are moved to the top of their containing scope during compile phase. For example:
console.log(x); // undefined
var x = 5;
Because var x is hoisted, but assignment stays where it is. let and const are hoisted differently (in a temporal dead zone) so accessing them before their declaration throws a ReferenceError.
4. What are closures in JavaScript?
Answer: A closure is when an inner function retains access to its outer function’s scope even after the outer function has returned. Example:
function outer() {
  let count = 0;
  return function inner() {
    count++;
    console.log(count);
  }
}
const fn = outer();
fn(); // 1
fn(); // 2
Here inner() closes over count variable—this is a closure.
5. What is the difference between == and ===?
Answer:
•	== is equality with type conversion (coercion). Example: 5 == '5' is true.
•	=== is strict equality—both value and type must match. Example: 5 === '5' is false.
It’s best to use === to avoid unexpected coercions.
6. What is the DOM and how do you manipulate it with JavaScript?
Answer: The DOM (Document Object Model) is the tree like representation of the HTML document elements and their properties. With JavaScript you can manipulate it:
const btn = document.getElementById("myBtn");
btn.addEventListener("click", () => {
  document.getElementById("text").textContent = "Clicked!";
});
You can select elements (getElementById, querySelector), change attributes, add/remove elements, listen to events, etc.
7. Explain event bubbling and event delegation.
Answer:
•	Event bubbling: When an event occurs on an element, it first runs its handlers, then the event moves up (bubbles) to ancestor elements, triggering their handlers.
•	Event delegation: Instead of attaching handler to many child elements, you attach a handler to a parent and use the event target to determine which child triggered the event. This improves performance and code brevity.
8. What is the difference between synchronous and asynchronous code?
Answer:
•	Synchronous code runs line by line, blocking further execution until current line finishes.
•	Asynchronous code allows other work to happen while waiting (for I/O, network). In JS you use callbacks, promises, async/await to handle asynchronous behavior.
9. What are Promises? How is async/await used?
Answer:
A Promise is an object that represents eventual completion or failure of asynchronous operation. Example:
const promise = new Promise((resolve, reject) => {
  setTimeout(() => resolve("done"), 1000);
});
promise.then(result => console.log(result));
async/await provides syntactic sugar:
async function getData() {
  try {
    const result = await fetch("/api");
    const data = await result.json();
    console.log(data);
  } catch (err) {
    console.error(err);
  }
}
It makes asynchronous code look synchronous.
10. What are arrow functions? How are they different from regular functions?
Answer:
Arrow functions (()=>{}) are shorthand syntax introduced in ES6. Differences:
•	They don’t have their own this, arguments, super, or new.target context—they use the surrounding lexical this.
•	They cannot be used as constructors (you can’t use new ArrowFunc()).
Example:
const add = (a,b) => a + b;

-Answer: It is a distributed version control system (VCS) designed to track changes in files, especially source code, and facilitate collaboration among developers.
-GitHub is a web-based platform that hosts Git repositories, providing developers with tools for version control and collaboration.  GitHub uses Git, a distributed version control system that tracks changes to your code. Git allows multiple developers to work on a project simultaneously without overwriting each other's changes. It keeps a history of all changes, making it easy to revert to previous versions if needed.

Answer: Media queries allow CSS to apply styles conditionally based on viewport size or device characteristics. Example:
@media (max-width: 768px) {
  .sidebar { display: none; }
}
Responsive pages use flexible layouts (flex, grid), relative units (%, em, rem), and breakpoints to adjust for mobile/desktop.
8. What’s the difference between em, rem, %, and px?
Answer:
•	px: absolute pixel size.
•	%: relative to parent element’s size (for width/height) or font size for text context.
•	em: relative to the font size of the element itself.
•	rem: relative to the font size of the root element (usually <html>).
Relative units scale better across devices.
9. How do Flexbox and Grid differ? When would you use each?
Answer:
•	Flexbox is one dimensional (row or column) layout—good for aligning items along one axis.
•	Grid is two dimensional (rows and columns)—good for complex layouts where alignment across both axes matters.
Use Flexbox for things like a nav bar or card row; Grid for full page layout or irregular grid needs.
10. What is the z index and how does stacking context work?
Answer: z index controls stacking order (which elements appear on top) when elements overlap and have a position value other than static. Stacking context is created by position, opacity, transform, and other properties—only within the same context z index compares. Understanding this helps avoid unexpected layering issues.

🟡 JavaScript Questions & Answers
1. What is JavaScript and how is it different from Java?
Answer: JavaScript is a scripting language that runs in the browser (and now on servers with Node.js). It’s dynamically typed and prototype based. Java is a statically typed, object oriented language compiled to bytecode running on a JVM. Despite the similar names, they differ fundamentally.
2. What are var, let, and const?
Answer:
•	var declares a function scoped or globally scoped variable. It is hoisted (initialization to undefined).
•	let declares a block scoped variable (within {}) and is not hoisted in the same way.
•	const declares a block scoped constant (cannot be reassigned). Its contents (for objects/arrays) may be modified, but the binding cannot be reassigned.
3. What is hoisting in JavaScript?
Answer: Hoisting is JavaScript’s behavior where declarations (var, function) are moved to the top of their containing scope during compile phase. For example:
console.log(x); // undefined
var x = 5;
Because var x is hoisted, but assignment stays where it is. let and const are hoisted differently (in a temporal dead zone) so accessing them before their declaration throws a ReferenceError.
4. What are closures in JavaScript?
Answer: A closure is when an inner function retains access to its outer function’s scope even after the outer function has returned. Example:
function outer() {
  let count = 0;
  return function inner() {
    count++;
    console.log(count);
  }
}
const fn = outer();
fn(); // 1
fn(); // 2
Here inner() closes over count variable—this is a closure.
5. What is the difference between == and ===?
Answer:
•	== is equality with type conversion (coercion). Example: 5 == '5' is true.
•	=== is strict equality—both value and type must match. Example: 5 === '5' is false.
It’s best to use === to avoid unexpected coercions.
6. What is the DOM and how do you manipulate it with JavaScript?
Answer: The DOM (Document Object Model) is the tree like representation of the HTML document elements and their properties. With JavaScript you can manipulate it:
const btn = document.getElementById("myBtn");
btn.addEventListener("click", () => {
  document.getElementById("text").textContent = "Clicked!";
});
You can select elements (getElementById, querySelector), change attributes, add/remove elements, listen to events, etc.
7. Explain event bubbling and event delegation.
Answer:
•	Event bubbling: When an event occurs on an element, it first runs its handlers, then the event moves up (bubbles) to ancestor elements, triggering their handlers.
•	Event delegation: Instead of attaching handler to many child elements, you attach a handler to a parent and use the event target to determine which child triggered the event. This improves performance and code brevity.
8. What is the difference between synchronous and asynchronous code?
Answer:
•	Synchronous code runs line by line, blocking further execution until current line finishes.
•	Asynchronous code allows other work to happen while waiting (for I/O, network). In JS you use callbacks, promises, async/await to handle asynchronous behavior.
9. What are Promises? How is async/await used?
Answer:
A Promise is an object that represents eventual completion or failure of asynchronous operation. Example:
const promise = new Promise((resolve, reject) => {
  setTimeout(() => resolve("done"), 1000);
});
promise.then(result => console.log(result));
async/await provides syntactic sugar:
async function getData() {
  try {
    const result = await fetch("/api");
    const data = await result.json();
    console.log(data);
  } catch (err) {
    console.error(err);
  }
}
It makes asynchronous code look synchronous.
10. What are arrow functions? How are they different from regular functions?
Answer:
Arrow functions (()=>{}) are shorthand syntax introduced in ES6. Differences:
•	They don’t have their own this, arguments, super, or new.target context—they use the surrounding lexical this.
•	They cannot be used as constructors (you can’t use new ArrowFunc()).
Example:
const add = (a,b) => a + b;
